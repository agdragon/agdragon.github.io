mac下nginx的安装与使用

2016-10-14 10:57:14

Nginx安装

先安装PCRE库(注：nginx rewrite依赖该库)

可以在这里下载最新版（这里使用的是8.33的版本），然后在终端执行下面的命令。

cd ~/Download
tar xvzf pcre-8.33.tar.gz
cd pcre-8.33
sudo ./configure --prefix=/usr/local
sudo make
sudo make install

终端执行：

brew search nginx
brew install nginx

安装完以后，可以在终端输出的信息里看到一些配置路径：

/usr/local/etc/nginx/nginx.conf （配置文件路径）

/usr/local/var/www （服务器默认路径）

/usr/local/Cellar/nginx/1.6.2  （貌似是安装路径）

Nginx常用命令

nginx -V 查看版本，以及配置文件地址
nginx -v 查看版本
nginx -c filename 指定配置文件
nginx -h 帮助

启动 Nginx 

sudo nginx

Nginx 默认8080端口，这时已经可以在浏览器访问了：

localhost:8080

会有一个默认欢迎界面。

重新加载配置|重启|停止|退出 nginx

nginx -s reload|reopen|stop|quit

Nginx 重载配置

sudo nginx -s reload

上述是采用向 Nginx 发送信号的方式，或者使用：

service nginx reload

备注: 做个软连接 ln -s /usr/local/sbin/nginx /usr/bin/nginx

停止并退出 Nginx

sudo nginx -s stop
sudo nginx -s quit

-s都是采用向 Nginx 发送信号的方式。

测试配置是否有语法错误

nginx -t

指定配置文件

sudo nginx -c /usr/local/nginx/conf/nginx.conf

-c表示configuration，指定配置文件。

查看 Nginx 版本

有两种可以查看 Nginx 的版本信息的参数。第一种如下：

nginx -v

另一种显示的是详细的版本信息：

nginx -V

检查配置文件是否正确

nginx -t

如果出现如上的提示信息，表示没有访问错误日志文件和进程，可以sudo（super user do）一下：

sudo nginx -t

如果显示如上，则表示配置文件正确。否则，会有相关提示。

显示帮助信息

nginx -h

或者：

nginx -?

[root@centos-1-22 sbin]# ./nginx -h
nginx version: nginx/1.0.15
Usage: nginx [-?hvVtq] [-s signal] [-c filename] [-p prefix] [-g directives]

Options:
  -?,-h         : this help
  -v            : show version and exit
  -V            : show version and configure options then exit
  -t            : test configuration and exit
  -q            : suppress non-error messages during configuration testing
  -s signal     : send signal to a master process: stop, quit, reopen, reload
  -p prefix     : set prefix path (default: /usr/local/nginx/)
  -c filename   : set configuration file (default: conf/nginx.conf)
  -g directives : set global directives out of configuration fil

以上这些涵盖了 Nginx 日常维护的所有基本操作
